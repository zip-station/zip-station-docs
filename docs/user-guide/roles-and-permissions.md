# Roles and Permissions

Zip Station uses role-based access control to manage what each team member can do.

## Owner Role

Every workspace has one **Owner**. The Owner has full access to everything and cannot be deleted. This role is automatically assigned to whoever creates the workspace.

## Custom Roles

Create custom roles to control access for your team.

### Creating a Role

1. Go to **Settings** > **Roles**
2. Click **New Role**
3. Enter a role name (e.g., "Support Agent", "Manager")
4. Check the permissions this role should have

### Permission Categories

Permissions are grouped by feature:

- **Tickets** — View, create, reply, assign, change status, delete
- **Intake** — View, approve, deny, manage rules
- **Customers** — View, edit, delete
- **Canned Responses** — View, create, edit, delete
- **Projects** — View, create, edit settings
- **Settings** — View, edit company settings, manage members
- **Roles** — View, create, edit, delete roles

## Assigning Roles

Roles can be assigned at two levels:

- **Company-wide** — The role applies across all projects
- **Project-level** — The role only applies within a specific project

Go to **Settings** > **Team Members**, select a member, and assign their role. You can give different roles for different projects if needed.
